# 🤖 Autonomous AI Workforce Agent

> An AI-powered multi-agent system that thinks, plans, and executes tasks like a junior employee, not just a chatbot.

Built using Python, Groq (LLaMA 3.3 70B), ChromaDB RAG, and Streamlit.

---

## 🧠 System Architecture

```
User Goal
    ↓
Intent Detector (LLM Semantic Classification)
    ↓
┌──────────────────────────────────────────────────┐
│                  AGENT PIPELINE                  │
│                                                  │
│   Planner Agent → Executor Agent                 │
│        ↓               ↓                        │
│   Break goal       Route to Tools:               │
│   into steps       • Task Manager (functional)   │
│                    • RAG Retriever (functional)   │
│                    • HR Action Tool (functional)  │
│                    • Email / Doc / Calendar       │
│        ↓               ↓                        │
│        └──→  Reflector Agent ←──────────────┘   │
│               Evaluate + Self-Correct            │
└──────────────────────────────────────────────────┘
    ↓
Memory Store (JSON) + State Manager (source of truth)
```

---

## ✨ Key Features

- **Goal Understanding** — LLM breaks any natural language goal into actionable steps
- **Plan → Execute → Reflect Loop** — true autonomous agent behavior with self-correction
- **Multi-Tool System** — Task Manager, RAG Retriever, HR Tool, Email, Doc, Calendar tools
- **RAG Knowledge Base** — retrieves company policies using ChromaDB + Groq summarization
- **Persistent Memory** — remembers past goals and tasks across sessions via JSON
- **State Manager** — tracks execution state as single source of truth for the Reflector
- **Self-Correction Loop** — Reflector detects real gaps and re-plans automatically (max 2 retries)
- **Semantic Intent Detection** — LLM understands user intent, not just keyword matching
- **Streamlit Web UI** — interactive dashboard with memory panel and execution visualization

## ⚙️ Setup Instructions

### 1. Clone the Repository
```bash
git clone https://github.com/YOUR_USERNAME/workforce-agent.git
cd workforce-agent
```

### 2. Create Virtual Environment
```bash
python -m venv venv

# Windows
venv\Scripts\activate

# Mac / Linux
source venv/bin/activate
```

### 3. Install Dependencies
```bash
pip install -r requirements.txt
```

### 4. Set Up Environment Variables
Create a `.env` file in the root folder:
```
GROQ_API_KEY=your_groq_api_key_here
```
Get your free Groq API key at: https://console.groq.com

### 5. Run CLI Version
```bash
python main.py
```

### 6. Run Web UI
```bash
python -m streamlit run app.py
```
Opens at: **http://localhost:8501**

## 🤖 Agent Descriptions

### Planner Agent (`planner.py`)
Receives the user goal and uses LLaMA 3.3 70B via Groq to decompose it into 3-6 concrete, actionable steps. Returns a structured JSON task list.

### Executor Agent (`executor.py`)
Iterates through all planned tasks. For each task, calls the Classifier to determine task type, then routes to the correct tool for execution.

### Classifier (`classifier.py`)
Uses LLaMA 3.3 70B to semantically classify each task into one of: `communication`, `document_work`, `meeting_management`, `research`, `planning`, `hr_task`, `data_processing`, or `general`.

### Reflector Agent (`reflector.py`)
After execution completes, evaluates whether the goal was truly achieved using real execution state — not text parsing. If gaps are found, triggers a re-planning loop with a maximum of 2 retries.

### Memory Store (`memory/memory_store.py`)
Persists current goal, task list, interaction history, and all past goals to a JSON file. Enables context continuity — the agent can recall what it was working on in previous sessions.

### State Manager (`state_manager.py`)
Tracks execution state as key-value pairs (e.g. `leave_applied: true`, `email_sent: true`). Serves as the single source of truth for the Reflector, preventing false gap detection from ambiguous text output.
